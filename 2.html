HTML <- 웹을 만들기 위한 기초, 쉬우나 가장 중요함.
HTML태그 -> <태그이름> 으로 사용함.
-> 태그 사이에 또다른 태그를 집어넣을때, 바깥쪽 태그를 부모 태그, 안쪽 태그를 자식 태그 (계속 반복)
- !doctype html : 웹페이지가 HTML로 짜여져있음을 알려주는 태그
- strong : 태그 사이에 있는 글자를 진하게 바꿈
- u : 태그 사이에 있는 글자에 밑줄을 그음
- h1 : 가장 큰 제목 태그
- h(n) : h1부터 점점 작아지는 제목 태그
- br : 줄바꿈 태그 (닫지 않아도 됨)
- p : 단락 태그
- img : 이미지 태그, src 속성을 통해 이미지 위치를 지정해야함 (width 속성으로 크기 조절 가능)
- a : 링크 태그, href 속성을 통해 위치 지정 가능
앞쪽 태그 뒤에 Attribute(속성)인 style="??" 를 통해 CSS 스타일 적용 가능

우리가 검색창에 주소를 (ex/index.html)적어서 나온 이 웹페이지는 바로 htdocs라는 디렉토리 밑에 있는 index-html파일이다.
한 대의 컴퓨터에 bitnami가 만든 htdocs라는 디렉토리 안에 index.html 이라는 파일이 저장되어 있다.
그리고 우리 컴퓨터에 웹브라우저와 웹서저가 모두 한 대의 컴퓨터에 설치된 상태이다.
ex)http:/ /127.0.0.1/index.html
여기서 127.0.0.1은 internet protocaol Address = IP주소
웹브라우저가 설치되어있는 컴퓨터를 가리키는 아주 특수한 주소임(약속되어 있는 주소)
웹브라우저가 설치되어 있는 각자의 컴퓨터의 웹서버를 가리킨다.
index.html을주면 웹브라우저가 자신의 컴퓨터에 설치되어 있는 웹서버에 접속해서 “index.html”파일을 원해“라고 이야기를 하는 것. 그럼 웹서버는 htdocs라는 디렉토리에서 파일을 찾도록 설정이 기본적으로 되어 있는 상태.
그럼 웹서버는 설정이 외어 있는 대로 htdocs라는 디렉토리애ㅐ서 index.html이라는 파일을 읽어서 그 파일을 웹브라우저에게 전송해주면 웹브라우저는 그것을 해석해서 화면에 표시
Q.파일 열기를 통해서 웹페이지를 보는 것과 주소창에 주소를 입력해서 웹페이지를 보는 것의 차이는?
우리사 주소를 h쎄://로 입력했다는 것은 이 컴퓨터에 웹브라우저와 웹서버가 있을 때 웹브라우저가 웹서버에게 요청하는 것이다.
그럼 웹섭이 index.html파일을 열어서 웹브라우저에게 전송해주는 것이다.(한대의 컴퓨터 안에서!)
but file:// 로 시작하는 주소를 입력하면 이 컴퓨터 안에서는 웹서버가 이 과정에 개입하지 않는다. 웹브라우저가 직접 index.html 파일을 읽어서 화면에 열어주는 것이다.
웹브라우저와 웹서버가 서로 통신할 때 사용하는 통신규약인 Hyper Text Transfer Protocol을 이용해서 데이터를 가져오는 것이다. 웹페이지를 웹브라우저와 웹서버에 서로 주고받기 위한 약속
암튼! 이 두가지 방식은 결과적으로는 사용자에게 전혀 차이가 없지만 그것은 우리가 한 대의 컴퓨터 안에 웹브라우저와 웹서버가 동시에 설치되어 있기 때문이다. 따라서 서로 다른 컴퓨터에 있는 웹브라우저와 웹서버가 통신하기 위해서는 반드시 http://를 사용해야한다.
